# GOD CRM Bot v3.0 — Multi-User

Один Telegram-бот для всей команды с интеграцией GOD CRM.
Мульти-юзер: каждый пользователь привязан к своему CRM-аккаунту.

## Мульти-юзер система

### Как это работает

1. **Админ** (GERATRON) запускает бота с `ADMIN_TELEGRAM_ID` в .env
2. Незнакомый юзер пишет боту — бот показывает его **Telegram ID**
3. Админ добавляет юзера: `/adduser <telegram_id> <имя> <роль>`
4. Юзер получает доступ ко всем командам
5. У каждого юзера свой CRM User ID — данные не пересекаются

### Роли

| Роль | Описание |
|------|----------|
| `admin` | Полный доступ + управление юзерами |
| `partner` | Все команды, без админских |
| `viewer` | Все команды, без админских |

### Админ-команды

| Команда | Описание |
|---------|----------|
| `/adduser <tg_id> <имя> <роль>` | Добавить юзера |
| `/adduser <tg_id> <имя> <роль> <crm_id> <email>` | С CRM привязкой |
| `/removeuser <tg_id>` | Удалить юзера |
| `/users` | Список всех юзеров |
| `/setcrm <tg_id> <crm_id> <email>` | Привязать CRM аккаунт |
| `/settable <tg_id> <sprint\|health> <table_id>` | Назначить таблицу |

## Быстрый старт

### Шаг 1: Получи токен от BotFather

1. Открой Telegram, найди **@BotFather** (https://t.me/BotFather)
2. Напиши `/newbot`
3. Придумай имя (например `GOD CRM Bot`)
4. Придумай username (например `godcrm_bot`)
5. Скопируй токен (вида `7123456789:AAHx1234...`)

> **Никому не показывай токен!**

### Шаг 2: Настрой конфиг

```bash
cd bots/nikitron-bot
cp .env.example .env
```

Отредактируй `.env`:
- `TELEGRAM_BOT_TOKEN` — токен от BotFather (**обязательно!**)
- `ADMIN_TELEGRAM_ID` — твой Telegram ID (узнать: напиши боту `/whoami`)
- `CRM_JWT_SECRET` — JWT secret для CRM (для CRM-команд)
- `CRM_USER_ID` — твой CRM User ID
- `CRM_USER_EMAIL` — твой CRM email

### Шаг 3: Установи и запусти

```bash
npm install
npm start
```

### Шаг 4: Добавь себя и других

1. Напиши боту `/whoami` — узнай свой Telegram ID
2. Добавь свой `ADMIN_TELEGRAM_ID` в `.env`, перезапусти бота
3. Добавь других: `/adduser 123456789 NIKITRON partner`
4. Настрой CRM: `/setcrm 123456789 5 nikita@hltrn.cc`

## Режимы работы

### Без CRM (базовый)
Бот работает и без CRM! Fun-команды (`/dice`, `/joke`, `/time`) работают всегда.
CRM-команды покажут предупреждение.

### С CRM (полный)
Когда настроен `CRM_JWT_SECRET`:
- Спринт, задачи, здоровье — всё берётся из CRM
- Чат с AI-агентами прямо из Telegram
- Каждый юзер видит СВОИ данные (свой CRM User ID)

---

## Защита от ошибок

Бот **НЕ падает** при ошибках:
- Все запросы к CRM обёрнуты в try/catch
- Автоматический retry при 5xx ошибках (до 3 попыток)
- Таймаут 15 секунд на CRM-запросы
- Если Markdown не парсится — отправляет без форматирования
- `uncaughtException` и `unhandledRejection` ловятся глобально
- Каждое обновление обрабатывается изолированно

---

## Хранение данных

Юзеры сохраняются в `users.json` (рядом с bot.js).
Файл создаётся автоматически при первом `/adduser`.
При перезапуске бота все юзеры сохраняются.
